HumanBound Relay Proxy

Local relay architecture enabling HumanBound cloud execution against a network-restricted internal API at Lloyds Banking Group.

Version 1.0 | February 2026 | FCA-regulated environment

The Problem

HumanBound operates as a cloud-hosted execution engine. It generates payloads, sends them to the target API, observes responses, and adapts in real-time — the intelligence is in the feedback loop. It cannot be reduced to a job queue.

The target API at Lloyds is not internet-facing. It sits behind private DNS, IP restriction, and proxy enforcement. HumanBound's cloud infrastructure cannot reach it.

Your laptop, when on the Lloyds network, can reach the API. The solution: make your laptop a transparent relay so HumanBound cloud believes it's talking directly to the API.

The relay proxy runs on your laptop inside the Lloyds network. It receives requests from HumanBound via a secure tunnel, forwards them to the internal API, and returns responses. HumanBound never touches the internal network directly — all traffic transits through your controlled, auditable relay.

Request Flow

HumanBound Cloud
Your Laptop (Relay)
Lloyds Internal API
1 HumanBound sends test payload → tunnel → relay proxy on localhost:8443
2 Relay logs the request (headers, body, timestamp) to audit file
3 Relay injects/refreshes JWT bearer token for internal API auth
4 Relay forwards request to internal API (rewriting Host, path as needed)
5 Internal API responds
6 Relay logs the response, returns it through tunnel to HumanBound
7 HumanBound analyses response, generates next adaptive payload → repeat

Tunnel Options

You need a way for HumanBound cloud to reach your laptop's relay proxy. The tunnel must be outbound-initiated from your laptop (no inbound firewall changes). Here are the realistic options ranked by bank-friendliness:

Method How it works SOC Risk Setup Bank-friendly
Cloudflare Tunnel Outbound HTTPS to Cloudflare edge. No open ports. Looks like normal web traffic. Low Simple Best
ngrok Outbound to ngrok's edge. Gives you a public URL. Medium Simple OK
SSH reverse tunnel SSH -R to your VPS. Expose relay port via VPS. Medium Moderate OK
WireGuard / Tailscale Encrypted mesh VPN. Laptop joins your private net. High Moderate Risky
Recommendation

Cloudflare Tunnel is the strongest option for a bank environment. It's outbound-only HTTPS on port 443, passes through corporate proxies transparently, and leaves no listening ports on your machine. You control the tunnel from a Cloudflare dashboard and can kill it instantly. If they ask what it is, it's a "secure application access tunnel" — the same technology Cloudflare sells to banks for zero-trust access.

Implementation: The Relay Proxy

This is the core component — a Python reverse proxy running on your laptop that receives HumanBound traffic, handles JWT auth, forwards to the internal API, and logs everything for FCA compliance.

relay_proxy.py Python 3.10+ 
#!/usr/bin/env python3
"""
HumanBound Relay Proxy
Sits on your laptop inside the Lloyds network.
Receives requests from HumanBound cloud via tunnel,
forwards to internal API, returns responses.
All requests/responses logged for FCA audit trail.
"""

import json, time, uuid, hashlib, logging, os, sys
from datetime import datetime, timezone
from pathlib import Path

try:
    from flask import Flask, request, Response
    import requests as http_client
except ImportError:
    print("pip install flask requests")
    sys.exit(1)

# ── Configuration ──────────────────────────────────────

TARGET_API_BASE    = os.getenv("TARGET_API", "https://internal-api.lloyds.com")
LISTEN_PORT        = int(os.getenv("RELAY_PORT", "8443"))
SHARED_SECRET      = os.getenv("RELAY_SECRET", "CHANGE_ME_TO_RANDOM_SECRET")
JWT_TOKEN          = os.getenv("JWT_TOKEN", "")
LOG_DIR            = Path(os.getenv("LOG_DIR", "./audit_logs"))
ALLOWED_PATHS      = os.getenv("ALLOWED_PATHS", "").split(",") if os.getenv("ALLOWED_PATHS") else []
RATE_LIMIT_RPS     = int(os.getenv("RATE_LIMIT", "10"))

# ── Setup ──────────────────────────────────────────────

app = Flask(__name__)
LOG_DIR.mkdir(parents=True, exist_ok=True)

# Structured audit logger — tamper-evident with chained hashes
audit_log_path = LOG_DIR / f"relay_audit_{datetime.now().strftime('%Y%m%d_%H%M%S')}.jsonl"
seq_counter = 0
prev_hash = "GENESIS"

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s"
)
log = logging.getLogger("relay")

# Simple rate limiter
request_times = []

# ── Audit Logging ──────────────────────────────────────

def audit_log(entry: dict):
    """Append a tamper-evident log entry with chained SHA-256 hashes."""
    global seq_counter, prev_hash
    seq_counter += 1
    entry["seq"] = seq_counter
    entry["timestamp"] = datetime.now(timezone.utc).isoformat()
    entry["prev_hash"] = prev_hash

    # Chain hash: SHA-256 of previous hash + current entry
    raw = json.dumps(entry, sort_keys=True)
    entry["hash"] = hashlib.sha256(
        f"{prev_hash}:{raw}".encode()
    ).hexdigest()
    prev_hash = entry["hash"]

    with open(audit_log_path, "a") as f:
        f.write(json.dumps(entry) + "\n")

# ── Rate Limiting ──────────────────────────────────────

def check_rate_limit() -> bool:
    now = time.time()
    request_times[:] = [t for t in request_times if now - t < 1.0]
    if len(request_times) >= RATE_LIMIT_RPS:
        return False
    request_times.append(now)
    return True

# ── Auth Verification ──────────────────────────────────

def verify_caller() -> bool:
    """Verify the request is from HumanBound via shared secret."""
    token = request.headers.get("X-Relay-Secret", "")
    return token == SHARED_SECRET

# ── Path Filtering ─────────────────────────────────────

def is_path_allowed(path: str) -> bool:
    """If ALLOWED_PATHS is set, only permit those prefixes."""
    if not ALLOWED_PATHS or ALLOWED_PATHS == [""]:
        return True
    return any(path.startswith(p.strip()) for p in ALLOWED_PATHS)

# ── Core Relay Handler ─────────────────────────────────

@app.route("/", defaults={"path": ""}, methods=[
    "GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS", "HEAD"
])
@app.route("/path:path>", methods=[
    "GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS", "HEAD"
])
def relay(path):
    request_id = str(uuid.uuid4())[:8]

    # ── Gate checks ──
    if not verify_caller():
        audit_log({"event": "AUTH_REJECTED", "id": request_id,
                   "src": request.remote_addr})
        return Response("Unauthorized", status=401)

    if not check_rate_limit():
        audit_log({"event": "RATE_LIMITED", "id": request_id})
        return Response("Rate limited", status=429)

    if not is_path_allowed(f"/{path}"):
        audit_log({"event": "PATH_BLOCKED", "id": request_id,
                   "path": path})
        return Response("Path not allowed", status=403)

    # ── Build target request ──
    target_url = f"{TARGET_API_BASE}/{path}"
    if request.query_string:
        target_url += f"?{request.query_string.decode()}"

    # Forward headers, strip relay-specific ones
    headers = {k: v for k, v in request.headers
               if k.lower() not in (
                   "host", "x-relay-secret",
                   "transfer-encoding"
               )}

    # Inject JWT if configured
    if JWT_TOKEN:
        headers["Authorization"] = f"Bearer {JWT_TOKEN}"

    body = request.get_data()

    # ── Log request ──
    audit_log({
        "event": "REQUEST",
        "id": request_id,
        "method": request.method,
        "target": target_url,
        "body_size": len(body),
        "body_preview": body[:500].decode("utf-8", errors="replace")
    })

    # ── Forward to internal API ──
    try:
        resp = http_client.request(
            method=request.method,
            url=target_url,
            headers=headers,
            data=body,
            timeout=30,
            verify=True,
            allow_redirects=False
        )
    except Exception as e:
        audit_log({
            "event": "TARGET_ERROR",
            "id": request_id,
            "error": str(e)
        })
        return Response(f"Relay error: {e}", status=502)

    # ── Log response ──
    audit_log({
        "event": "RESPONSE",
        "id": request_id,
        "status": resp.status_code,
        "body_size": len(resp.content),
        "body_preview": resp.content[:500].decode(
            "utf-8", errors="replace")
    })

    # ── Return to HumanBound ──
    excluded_headers = {"transfer-encoding", "content-encoding",
                        "content-length", "connection"}
    response_headers = {k: v for k, v in resp.headers.items()
                        if k.lower() not in excluded_headers}

    return Response(
        resp.content,
        status=resp.status_code,
        headers=response_headers
    )

# ── Health Check ───────────────────────────────────────

@app.route("/__relay/health")
def health():
    return {
        "status": "ok",
        "target": TARGET_API_BASE,
        "uptime_requests": seq_counter,
        "log_file": str(audit_log_path)
    }

# ── Status Page ────────────────────────────────────────

@app.route("/__relay/stats")
def stats():
    if not verify_caller():
        return Response("Unauthorized", status=401)
    return {
        "total_requests": seq_counter,
        "log_file": str(audit_log_path),
        "log_size_bytes": audit_log_path.stat().st_size
            if audit_log_path.exists() else 0,
        "target": TARGET_API_BASE,
        "rate_limit": RATE_LIMIT_RPS,
        "path_filter": ALLOWED_PATHS or "ALL"
    }

# ── Entry Point ────────────────────────────────────────

if __name__ == "__main__":
    log.info(f"╔══════════════════════════════════════════╗")
    log.info(f"║  HumanBound Relay Proxy                  ║")
    log.info(f"║  Target: {TARGET_API_BASE:<30} ║")
    log.info(f"║  Port:   {LISTEN_PORT:<30} ║")
    log.info(f"║  Log:    {str(audit_log_path):<30} ║")
    log.info(f"╚══════════════════════════════════════════╝")

    if SHARED_SECRET == "CHANGE_ME_TO_RANDOM_SECRET":
        log.warning("⚠ USING DEFAULT SECRET — set RELAY_SECRET env var!")

    app.run(
        host="127.0.0.1",   # localhost only — tunnel handles exposure
        port=LISTEN_PORT,
        debug=False
    )

Running It

Step 1 — Install dependencies on your laptop

Terminal bash 
pip install flask requests

Step 2 — Configure and start the relay

Terminal bash 
# Set your target API and a strong shared secret
export TARGET_API="https://internal-api.lloyds.com/v1"
export RELAY_SECRET="$(openssl rand -hex 32)"
export JWT_TOKEN="your-jwt-token-here"
export RELAY_PORT="8443"

# Optional: restrict to specific API paths only
export ALLOWED_PATHS="/api/v1/chat,/api/v1/completions"

# Start the relay
python relay_proxy.py

Step 3 — Establish the tunnel (Cloudflare example)

Terminal (second tab) bash 
# Install cloudflared (if not already)
brew install cloudflared   # or download binary

# Quick tunnel — gives you a public URL instantly
cloudflared tunnel --url http://localhost:8443

# Output will show something like:
# https://random-name-here.trycloudflare.com
# ↑ Give THIS URL to HumanBound as the target API endpoint

Step 4 — Point HumanBound at the tunnel URL

In your HumanBound configuration, set the target API URL to the Cloudflare tunnel URL. HumanBound will send requests to that URL, which routes through the tunnel to your laptop's relay proxy, which forwards to the real internal API.

Important: JWT management

The relay injects the JWT token you provide via the JWT_TOKEN env var. When the token expires, you'll need to refresh it. For short-lived tokens, you should extend the relay to call the bank's token endpoint automatically — add a refresh_jwt() function that runs before token expiry.

HumanBound Configuration

When configuring HumanBound to test the API, you need to tell it to include the relay authentication header on every request:

HumanBound config yaml / json 
# Target URL is your Cloudflare tunnel, NOT the real API
target_url: "https://random-name-here.trycloudflare.com"

# Add relay auth header so the proxy accepts the request
custom_headers:
  X-Relay-Secret: "your-64-char-hex-secret"

# Do NOT set Authorization header — the relay injects it
# HumanBound sends payloads → tunnel → relay → real API
What HumanBound sees

From HumanBound's perspective, it's just testing an API at a normal HTTPS URL. It has no idea a relay exists. It sends requests, gets responses, and runs its adaptive test logic as normal. The relay is transparent.

Audit Trail & FCA Compliance

The relay produces a tamper-evident JSONL audit log with chained SHA-256 hashes. Every entry contains a hash of itself plus the previous entry's hash, forming a verifiable chain. If any line is modified or deleted, the chain breaks.

To verify log integrity after the engagement:

verify_audit.py Python 
import json, hashlib

prev = "GENESIS"
with open("audit_logs/relay_audit_20260227_091500.jsonl") as f:
    for i, line in enumerate(f, 1):
        entry = json.loads(line)
        stored_hash = entry.pop("hash")
        raw = json.dumps(entry, sort_keys=True)
        expected = hashlib.sha256(f"{prev}:{raw}".encode()).hexdigest()
        assert stored_hash == expected, f"CHAIN BROKEN at seq {i}"
        prev = stored_hash

print(f"✓ All {i} entries verified. Chain intact.")

Security Hardening

  • 1
    Localhost binding only. The relay binds to 127.0.0.1 — it never listens on the network interface. Only the tunnel can reach it. Nobody on the Lloyds network can hit your relay directly.
  • 2
    Shared secret authentication. Every request must carry the X-Relay-Secret header. Without it, the relay returns 401. This prevents anyone who discovers the tunnel URL from using it.
  • 3
    Path allowlisting. Set ALLOWED_PATHS to restrict which API endpoints HumanBound can hit. If the target API has admin endpoints you shouldn't touch, block them here.
  • 4
    Rate limiting. Default 10 requests/second. Prevents runaway test loops from hammering the internal API and triggering production alerts.
  • 5
    Tamper-evident logging. Chained hashes mean you can prove no log entries were added, removed, or modified after the fact. Critical for FCA evidence.
  • 6
    Kill switch. Close the terminal running cloudflared and the tunnel dies instantly. No persistent access. No cleanup needed.

All API calls originate from within your network. The relay maintains a tamper-evident audit log of every request and response for post-engagement review. No inbound connections are opened. The relay can be terminated instantly and leaves no persistent access."

Honest Risk Assessment

Risk Severity Mitigation
SOC detects tunnel traffic Medium Pre-notify SOC. Cloudflare traffic on 443 looks like normal HTTPS. Provide them the tunnel hostname so they can whitelist monitoring.
Tunnel URL leaked Low Shared secret means leaked URL is useless without the header. Tunnel URLs are random and short-lived.
JWT expires mid-test Medium Extend relay with auto-refresh. Or use long-lived service account token for testing period.
Relay crashes / laptop sleeps Low HumanBound gets connection errors, pauses. Restart relay, resume. No data loss — log is append-only.
Bank policy prohibits tunnels High This is the real blocker. If they say no tunnels, you fall back to Option 3 (manual execution). Confirm policy before building.
Critical: Get approval first

Do not deploy this without written approval from the Lloyds engagement lead. Even though it's technically low-risk, deploying an unapproved tunnel in a bank will end the engagement. Frame it properly, get it signed off, then deploy.
